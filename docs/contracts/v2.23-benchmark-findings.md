# ShadowBid V2.23 Benchmark Findings

## Status

In progress, but the `USDCx` private-first path is currently paused.

This file is the running report for the `v2.23` Phase 1 feasibility spike and now records the current no-go decision for self-built `get_credentials` proof generation.

It is generated from benchmark artifacts under `contracts/benchmarks/v223-private-usdcx/`.

Last generated: `2026-04-03T16:57:21.182Z`

## Purpose

This report converts raw benchmark artifacts into a release decision for `v2.23`.

It should answer:

- does the tested private funding path hide the bid amount from the public path we are replacing?
- are proving time and fee cost acceptable for a user-facing auction flow?
- does the tested custody model preserve refund and winner settlement semantics?
- should the team continue, redesign, or stop before freezing the `v2.23` ABI?

## Current Result

Local sidecar execution evidence now exists for 5/5 benchmark scenarios.
The imported stablecoin rail still exposes the right private transfer primitives, but the repo cannot reproduce a usable `get_credentials` proof from public mappings alone.
That makes the `USDCx` private-first funding path a current no-go for `v2.23` release planning unless an external proof source exists.

Current recommendation:

- pause the `USDCx` private-first funding spike
- keep the `v2.23` ABI unfrozen
- treat current evidence as sidecar-readiness only, not private-escrow proof
- only reopen this path if a wallet, vendor, issuer, or off-chain proof builder can supply raw sibling fields or a usable `Credentials` record

## Decision Section

- live testnet state and the latest `get_creds` analysis say the current blocker is: the proof payload cannot be reconstructed from public mappings alone
- pause `v2.23` as a `USDCx` private-first release target

- only reopen this track when a real external proof source is demonstrated end to end
