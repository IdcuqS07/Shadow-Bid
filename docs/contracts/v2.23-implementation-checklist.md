# ShadowBid V2.23 Private Escrow Implementation Checklist

## Purpose

This checklist turns the `v2.23` private escrow proposal into concrete repo work.

Primary references:

- `docs/contracts/v2.23-private-escrow-proposal.md`
- `docs/contracts/v2.23-phase-1-feasibility-spike.md`
- `docs/contracts/privacy-remediation-roadmap.md`

Current status:

- this checklist is now conditional
- use it only if the `USDCx` private-first path is reopened after a real external `get_credentials` proof source is demonstrated

## Scope Assumptions

If reopened, this checklist assumes the `v2.23` MVP is:

- `USDCx` private-first
- honest about `ALEO` and `USAD` remaining on public or non-private paths unless explicitly upgraded later
- focused on hiding bid amounts in the funding path, not just in public mappings
- designed to preserve the `v2.22` auction lifecycle as much as possible

## Recommended Rollout Order

1. freeze the product scope and privacy claims
2. build a private `USDCx` feasibility spike plus benchmark harness
3. implement utility sidecar programs for record and credential setup
4. lock the `v2.23` auction ABI only after the private funding path is proven
5. implement the private auction path and settlement/refund behavior
6. update frontend, wallet UX, ops tooling, and docs
7. run testnet validation on privacy, refund, and winner settlement paths
8. deploy `v2.23`
9. cut UI traffic only to the paths that are actually privacy-ready
